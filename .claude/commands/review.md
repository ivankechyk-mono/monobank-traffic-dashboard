# /review

Прочитай `planning/SPEC.md`, `planning/CONSTITUTION.md` і `planning/STATE.md`.

Визнач яка задача щойно була реалізована (остання в STATE.md або запитай у користувача).

Потім проаналізуй змінені файли по чотирьох критеріях:

## 1. Відповідність spec і acceptance scenarios
- Чи реалізовано саме те що описано в SPEC.md для поточної story?
- Чи виконуються acceptance scenarios Given/When/Then?

## 2. Відповідність конституції (CONSTITUTION.md)
- Чи немає secrets у коді?
- Чи дотримано шарування connectors/transforms/loaders?
- Чи всі Google API scopes readonly?
- Чи немає SQLite або іншої забороненої БД?

## 3. Якість і simplicity
- Чи є дублювання коду яке можна прибрати?
- Чи є зайва складність?
- Чи зрозумілі назви?

## 4. Edge-кейси зі SPEC.md
- Чи оброблено edge cases (GSC затримка, Unknown product, нулі без розривів)?
